# 后端启动说明

## 环境要求

- **JDK 17+**（已安装）
- **Maven 3.6+**（仅首次打包和修改代码后需要，日常运行不需要）
- 无需安装 MySQL（项目内嵌 H2 数据库）

## 快速启动

### 方式一：双击启动脚本（推荐）

直接双击运行 **启动后端.bat**：

```
启动后端.bat
```

脚本会自动处理以下步骤：

| 步骤 | 操作 |
|---|---|
| 1 | 检测 JDK 是否安装 |
| 2 | 检查端口 8080，被占用则自动杀掉对应进程 |
| 3 | 查找 Maven，优先使用系统 PATH 中的 mvn |
| 4 | 在新窗口中启动 `mvn spring-boot:run` |
| 5 | 等待 12 秒后自动验证端口是否已监听 |

> 启动后脚本会打开一个新窗口（标题 "XYZ Market Backend"）用于运行后端。原脚本窗口会等待 12 秒后检测端口是否启动成功，并给出中文提示。

### 手动关闭后端

双击运行 **关闭后端.bat**：

```
关闭后端.bat
```

脚本自动处理：
1. 查找并杀掉所有 xyz-market 相关 Java 进程（通过 wmic）
2. 释放端口 8080
3. 清理 H2 数据库残留锁文件
4. 验证端口确认已释放

> **建议**：每次关闭后端后、再次启动前，都运行一次 `关闭后端.bat`，避免端口残留问题。

### 方式二：命令行启动（Trae 终端内）

进入项目目录：

```bash
cd 后端代码目录
```

执行以下命令启动：

```bash
mvn spring-boot:run -D"spring-boot.run.jvmArguments=-Dfile.encoding=UTF-8"
```

项目将启动在 **http://localhost:8080**

### 方式三：使用 IntelliJ IDEA

1. 用 IntelliJ IDEA 打开 `后端代码` 目录
2. 先执行 **Build → Rebuild Project**（确保代码修改生效）
3. 找到主类 `com.example.xyzmarket.XYZMarketApplication`
4. 右键 → Run
5. 在 VM options 中添加：`-Dfile.encoding=UTF-8`

## 数据库说明

项目使用内嵌 H2 数据库，无需额外安装。

- **默认模式**：使用文件数据库 `data/xyz_market`（位于项目根目录），**数据会持久化保存**，重启后数据不丢失
- 每次启动时自动执行 `init-h2.sql` 初始化测试数据（如用户、商品、举报信息等）

### 重置数据库

如果需要清空所有数据重新开始，删除项目根目录下的数据库文件即可：

```powershell
# 删除数据库文件
Remove-Item -Force "data\xyz_market.mv.db" -ErrorAction SilentlyContinue
Remove-Item -Force "data\xyz_market.trace.db" -ErrorAction SilentlyContinue
```

> **注意**：数据库文件位置由 `application.yml` 中的 `spring.datasource.url` 决定（`jdbc:h2:file:./data/xyz_market`），相对于**运行目录**。使用脚本启动时运行目录是 `后端代码` 文件夹，所以数据库文件生成在 `后端代码/data/` 下。

### 查看数据库内容

启动后访问：http://localhost:8080/h2-console

- JDBC URL: `jdbc:h2:file:./data/xyz_market;MODE=MySQL;DATABASE_TO_LOWER=TRUE;NON_KEYWORDS=USER`
- User Name: `sa`
- Password: 留空

## 验证是否启动成功

打开浏览器访问 http://localhost:8080/api/item/list?page=1&size=10

如果返回 JSON 数据，说明启动成功。

## 常见问题

### 端口被占用（Address already in use: 8080）

使用 **启动后端.bat** 脚本，它会自动杀掉占用 8080 端口的进程并等待 3 秒确认端口已释放。

如果手动操作：

```powershell
netstat -ano | findstr ":8080 "
taskkill /F /PID 查到的PID
```

### 数据库报错（Column "xxx" not found）

修改了后端实体类（新增字段）后，**旧的数据库文件没有新列**。

**解决**：删除旧数据库文件，重新启动（init-h2.sql 会重新创建带新列的表）：

```powershell
# 先关闭后端，再删除数据库文件
Remove-Item -Force "data\xyz_market.mv.db" -ErrorAction SilentlyContinue
```

### 数据库文件被锁（The file is locked）

如果遇到 `The file is locked` 错误，说明数据库文件被另一个进程占用。

**解决**：
1. 运行 `关闭后端.bat` 杀掉所有 Java 进程
2. 如果仍锁住，删除数据库文件后重新启动

### 编译不生效（修改代码后仍然报旧错误）

使用了旧的编译缓存。执行：

```bash
mvn clean spring-boot:run
```

`clean` 会删除 `target` 目录，强制重新编译。

### 中文乱码问题

启动脚本已加入 UTF-8 编码参数。如果仍出现乱码，删除 `data` 文件夹后重新启动（之前可能已存入了错误编码的数据）。

### 一键登录账号

测试环境下，每次重启后端后，一键登录使用固定账号 `test_openid_fixed_user_001`（昵称类似"用户12345"），**且每次启动都会自动重置该账号的校园认证状态**（清空认证信息），方便测试认证流程。

一键登录账号预置的测试数据（商品、订单）不受影响，每次启动都会保留。

**其他测试账号（张三、李四、王五、赵六、孙七、管理员等）不受影响，固定留存。**

## 前端连接配置

前端微信小程序在 `前端代码/项目/utils/http-util.js` 中配置后端地址，默认连接 `http://127.0.0.1:8080`。如果后端端口有修改，需要同步修改此处。
