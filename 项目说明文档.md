# 校易助（XYZ Market）项目说明文档

---

## 一、项目概述

**校易助（XYZ Market）** 是一款面向高校学生的**校园二手交易平台**微信小程序。学生可以在平台上发布闲置物品、浏览购买他人商品、发布求购需求、进行校园认证、在线聊天沟通，以及使用管理系统对平台进行审核管理。

- **架构**：前后端分离
- **后端**：Spring Boot 3.2.5 + MyBatis + H2 内嵌数据库 + JWT 认证
- **前端**：微信原生小程序
- **开发语言**：Java 17（后端）、JavaScript（前端）
- **构建工具**：Maven（后端）、微信开发者工具（前端）

## 三、核心功能详解

### 3.1 用户系统

| 功能 | 说明 |
|------|------|
| **一键登录** | 用户点击后通过微信授权直接登录，无需注册。测试环境下自动使用固定账号 `test_openid_fixed_user_001`，每次重启后端重置其校园认证状态 |
| **校园认证** | 用户提交学号和校园邮箱进行身份认证，由管理系统审核（通过/驳回）。通过后获得"本校认证"标签，在首页、详情页、求购区、聊天卡片中显示 |
| **个人资料** | 支持修改昵称，查看信用分、学号、年级等信息 |
| **信用分制度** | 初始60分，完成交易加分，违规扣分。管理系统可手动调整信用分 |

### 3.2 商品模块

| 功能 | 说明 |
|------|------|
| **商品发布** | 填写标题、描述、分类、价格，可上传多张图片，勾选"线下自提"和"可置换"标签 |
| **商品列表** | 首页展示全部在售商品，支持分页加载 |
| **商品搜索** | 关键词模糊搜索，支持按价格/时间排序 |
| **商品详情** | 展示标题、图片、价格、描述、卖家信息、认证标签 |
| **标签系统** | 本校认证（基于后端认证数据）、线下自提、可置换 |
| **我的发布** | 按"全部/在售/已售出/已下架"分tab展示，支持点击售出、下架操作 |
| **售出流程** | 在"我的发布"点击"售出"后，自动在"我卖出的"中生成已完成订单记录 |
| **下架流程** | 下架商品后自动取消该商品所有待确认订单，订单状态显示"已下架" |

### 3.3 订单模块

| 功能 | 说明 |
|------|------|
| **下单购买** | 在商品详情页点击"立即购买"创建订单 |
| **我的订单** | 分为"我买到的"和"我卖出的"两个tab |
| **我买到的** | 按"全部/待确认/已完成"筛选，待确认订单可取消 |
| **我卖出的** | 查看所有卖出的订单，状态包含已完成、已下架等 |
| **订单状态** | 待确认 → 已完成 / 已取消 / 已下架 |

### 3.4 收藏模块

| 功能 | 说明 |
|------|------|
| **收藏商品** | 在商品详情页点击收藏 |
| **我的收藏** | 查看所有收藏的商品列表，支持取消收藏（点击红心） |

### 3.5 求购模块

| 功能 | 说明 |
|------|------|
| **求购广场** | "全部求购"tab展示所有求购信息，每个卡片显示"本校认证"标签 |
| **发布求购** | 填写标题、描述、分类、预算 |
| **我的求购** | 查看自己发布的求购，支持"取消求购"操作（有防重复点击保护） |
| **取消后处理** | 取消后"取消"和"联系TA"按钮均消失 |

### 3.6 聊天模块

| 功能 | 说明 |
|------|------|
| **聊天列表** | 显示所有会话，每个卡片显示"本校认证"标签（认证用户） |
| **发起聊天** | 从商品详情页/求购页点击"联系卖家/联系TA"创建会话 |
| **聊天详情** | 支持发送文字消息，本地存储模拟IM通信 |

### 3.7 消息通知模块

| 功能 | 说明 |
|------|------|
| **通知列表** | 消息页面"通知"tab，展示订单状态变更、认证审核结果等通知 |
| **未读提示** | 未读通知高亮显示 + 红点标识 |
| **已读标记** | 点击通知标记为已读，支持"全部已读" |
| **通知类型** | 订单通知（下单、售出、取消）和认证通知（通过、驳回） |

### 3.8 管理系统

| 功能 | 说明 |
|------|------|
| **入口** | "我的"页面 → "管理系统入口"，输入密码（默认 `admin123`）登录 |
| **管理员锁定** | 连续5次密码错误后锁定入口，每次重启后端重置尝试次数 |
| **认证审核** | 查看待审核认证申请，支持批准或驳回（填写驳回理由） |
| **用户管理** | 查看所有用户列表，调整用户信用分 |
| **商品管理** | 查看所有商品列表，支持封禁违规商品 |
| **举报管理** | 查看举报列表，支持批准或驳回举报 |

### 3.9 举报模块

| 功能 | 说明 |
|------|------|
| **举报商品** | 在商品详情页提交举报，选择举报类型和描述 |
| **举报约束** | 自己的商品会提示"这是您自己的商品"，不可举报 |

### 3.10 其他功能

| 功能 | 说明 |
|------|------|
| **文件上传** | 支持图片上传到服务器本地 `uploads/` 目录 |
| **清除登录** | "我的"页面底部可清除登录信息 |
| **下拉刷新** | 主要列表页面支持下拉刷新 |

## 四、数据库设计

| 表名 | 说明 | 关键字段 |
|------|------|----------|
| `user` | 用户表 | openid, nickname, student_id, is_certified, credit_score, role |
| `item` | 商品表 | title, price, seller_id, category_id, status, pickup, tradeable |
| `orders` | 订单表 | item_id, buyer_id, seller_id, status |
| `certification` | 认证表 | user_id, student_id, email, status, reject_reason |
| `favorite` | 收藏表 | user_id, item_id（唯一约束） |
| `report` | 举报表 | reporter_id, item_id, type, description, status |
| `wanted` | 求购表 | user_id, title, budget, category_id, status |
| `notification` | 通知表 | user_id, type, title, content, is_read |

数据库使用 H2 文件模式（`jdbc:h2:file:./data/xyz_market`），每次启动自动执行 `init-h2.sql` 初始化表结构和测试数据。

## 七、启动与部署

### 7.1 环境要求

- **JDK 17+**（需配置 JAVA_HOME 或 PATH）
- **Maven 3.6+**（或使用项目自带的 Maven Wrapper `mvnw`）
- **微信开发者工具**（稳定版）
- **Windows 操作系统**

### 7.2 启动后端

1. 双击 `后端代码/启动后端.bat`
2. 脚本自动检测 JDK、检查端口 8080、查找 Maven
3. 等待约 12 秒，看到"XXX Started XYZMarketApplication in X seconds"即启动成功
4. 访问 `http://localhost:8080/h2-console` 可查看数据库（JDBC URL: `jdbc:h2:file:./data/xyz_market`，用户名 `sa`，密码为空）

### 7.3 关闭后端

双击 `后端代码/关闭后端.bat`，自动结束所有相关 Java 进程并清理数据库锁文件。

### 7.4 启动前端

1. 打开微信开发者工具
2. 导入项目：选择 `前端代码/项目` 目录
3. 填写 AppID（可使用测试号）
4. 确保后端已启动在 `localhost:8080`
5. 点击"编译"即可预览

### 7.5 测试流程

1. 启动后端 → 启动前端
2. "我的"页面 → 点击"一键登录"
3. 即可浏览商品、发布、下单、聊天等全部功能
4. 管理系统入口密码：`admin123`

## 九、项目亮点

1. **完整的二手交易闭环**：浏览 → 下单 → 确认售出 → 订单记录
2. **校园认证体系**：学号+邮箱认证，认证标识贯穿全平台
3. **求购功能**：支持发布求购需求，促进供需匹配
4. **管理系统**：独立的审核后台，支持认证审核、商品封禁、举报处理、信用分管理
5. **通知系统**：订单状态变更和认证审核结果自动推送通知
6. **一键启动**：Windows 批处理脚本，自动检测环境、释放端口、启动服务
7. **测试数据完备**：预置 7 个用户、多个商品和订单，开箱即可进行全功能测试
8. **标签系统**：本校认证、线下自提、可置换，多维度展示商品特色
