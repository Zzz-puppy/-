# 校易助 - 测试说明

## 一、环境要求

测试前请确保电脑已安装以下软件：

| 软件 | 版本要求 | 下载地址 |
|---|---|---|
| JDK | 17+ | https://adoptium.net/zh-CN/temurin/releases/?version=17 |
| 微信开发者工具 | 最新版 | https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html |

> 启动后端只需 JDK，不需要安装 Maven。首次运行 `启动后端.bat` 会自动查找 Maven，如果找不到会提示安装。
>
> 安装 JDK 后，在命令行输入 `java -version` 能正常显示版本号，说明环境配置成功。

## 三、启动步骤

### 步骤 1：启动后端

1. 进入 `后端代码` 文件夹
2. **双击运行 `启动后端.bat`**
3. 脚本会自动检测 JDK → 释放端口 8080 → 查找 Maven → 在新窗口中启动后端
4. 等待原脚本窗口提示 `[OK] Backend is running on http://localhost:8080` 即启动成功
5. **两个窗口都不要关闭**（Maven 窗口 + 脚本状态窗口）

> 首次运行时 Maven 可能需要下载依赖，耗时稍长属正常。

### 步骤 2：关闭后端（可选）

如果需要重新启动或关闭后端：

1. **双击运行 `关闭后端.bat`**
2. 脚本会自动杀掉 Java 进程、释放端口 8080
3. 看到 `Cleanup complete` 提示后即可再次运行 `启动后端.bat`

> 建议：关闭后端后再次启动前，都先运行一次 `关闭后端.bat`，避免端口残留问题。

### 步骤 3：验证后端是否启动

打开浏览器访问：

```
http://localhost:8080/api/item/list?page=1&size=10
```

如果返回 JSON 数据，说明后端启动成功。

### 步骤 4：启动前端

1. 打开 **微信开发者工具**
2. 点击 `项目` → `导入项目`
3. 选择 `前端代码/项目` 文件夹
4. AppID 选择 `测试号`（或填写自己的小程序 AppID）
5. 点击 `确定` 导入
6. 如果页面报错 `ERR_CONNECTION_REFUSED`，确认后端已启动

### 步骤 5：真机调试（可选）

需要用手机真机测试时：

1. 微信开发者工具中点击"真机调试"
2. **务必先在工具中勾选**：`详情` → `本地设置` → `不校验合法域名、web-view（业务域名）、TLS 版本以及 HTTPS 证书`
3. 手机和电脑连接**同一 WiFi 网络**
4. 修改前端连接地址：打开 `前端代码/项目/utils/http-util.js`，将 `baseUrl` 改为电脑的局域网 IP
5. 用微信扫码即可在手机上测试

> 查看本机 IP：在命令行输入 `ipconfig`，找到 `IPv4 地址`。

## 四、常见问题

### 1. 提示"端口 8080 被占用"

**解决**：
- `启动后端.bat` 会自动杀掉占用进程，等待即可
- 如果仍失败，先运行 `关闭后端.bat` 再试

### 2. 前端提示"网络请求失败"

**解决**：
- 确认后端已启动（浏览器能访问 `http://localhost:8080/api/item/list`）
- 确认 `http-util.js` 中的 `baseUrl` 地址正确
- 确认电脑防火墙没有阻止 8080 端口
- 真机调试时确认手机和电脑在同一局域网

### 3. 数据库报错（Column not found）

**原因**：修改后端代码新增字段后，旧的数据库文件没有新列。

**解决**：
```powershell
# 先运行 关闭后端.bat，然后删除数据库文件
Remove-Item -Force "后端代码\data\xyz_market.mv.db" -ErrorAction SilentlyContinue
# 重新运行 启动后端.bat
```

### 4. 小程序展示异常

**解决**：
- 确认微信开发者工具已勾选"不校验合法域名..."
- 检查开发者工具控制台是否有报错信息

### 5. 一键登录账号

测试环境下，一键登录使用固定账号 `test_openid_fixed_user_001`。每次重启后端后，系统会自动清空该账号的校园认证信息，方便测试完整的认证流程。原有测试数据（商品、订单）不受影响。
